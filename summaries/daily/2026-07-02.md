## 全件

### エージェント ノードを使用してエージェントをワークフロー ステップとして呼び出す

- 原題: Microsoft Copilot Studio: Invoke agents as workflow steps with the agent node
- ソース: Microsoft 365 Roadmap
- 公開日: 2026/07/02
- リリース段階: In development
- 役割タグ: 作成者向け
- URL: https://www.microsoft.com/microsoft-365/roadmap?id=566998

エージェント ノードを使用すると、推論をワークフローに直接埋め込むことができます。これは、手動の引き継ぎが減り、処理時間が短縮され、追加の開発作業を行わずに微妙な決定を処理できるワークフローを意味し、チームがエンドツーエンドのプロセスをより多く自動化し、運用オーバーヘッドを削減できるようにします。

### Excel の Copilot を使用した複雑な Web 検索

- 原題: Microsoft Copilot (Microsoft 365): Complex web search with Copilot in Excel
- ソース: Microsoft 365 Roadmap
- 公開日: 2026/07/02
- リリース段階: In development
- 役割タグ: 作成者向け
- URL: https://www.microsoft.com/microsoft-365/roadmap?id=566872

Excel の Copilot は、並行調査を調整し、結果を検証し、必要に応じてギャップを再チェックするマルチエージェント検索システムにより、複雑な Web 検索の品質を向上させ、複数のソースにわたってより迅速かつ完全な回答を提供します。

### 資格情報の過剰共有を検出して安全な共有を強制する

- 原題: Microsoft Copilot Studio: Enforce safe sharing by detecting credential oversharing
- ソース: Microsoft 365 Roadmap
- 公開日: 2026/07/02
- リリース段階: In development
- 役割タグ: 管理者向け / 作成者向け
- URL: https://www.microsoft.com/microsoft-365/roadmap?id=566873

この機能は、組織がメーカーの資格情報などの安全でない ID に依存している場合にエージェントとフローが共有されるのを防ぎ、セキュリティ リスクを軽減するのに役立ちます。 ID の過剰共有を早期に検出し、公開および共有時に安全な共有ポリシーを適用することで、お客様は ID の漏洩や権限の漏洩を回避できます。

- 原題: Microsoft Teams: Improved request flows for apps and agents blocked by admins
- ソース: Microsoft 365 Roadmap

- リリース段階: Rolling out
